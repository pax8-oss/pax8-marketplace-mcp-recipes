# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Email: security@pax8.com

Include in your report:

- A description of the vulnerability
- Steps to reproduce
- Potential impact

## Response Timeline

| Milestone | Target |
|---|---|
| Acknowledgement | 2 business days |
| Initial assessment | 5 business days |
| Resolution or mitigation plan | Depends on severity |

## Scope

This repository contains Markdown documentation (a Claude Skill). The attack surface is limited to:

- Content that could mislead an AI assistant into making incorrect API calls
- Links that could redirect users to malicious destinations

Issues outside this scope (e.g., vulnerabilities in the Pax8 MCP Server itself) should be reported directly to security@pax8.com with "Pax8 MCP" in the subject line.

## Credit

We will acknowledge researchers who report valid vulnerabilities, unless they request anonymity.
